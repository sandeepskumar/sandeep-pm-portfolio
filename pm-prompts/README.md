# 📋 PM Prompt Library

A curated collection of prompts I use regularly for high-quality, structured product management output. Organized by workflow area.

---

## 1. PRD (Product Requirements Document)

```
You are a senior product manager. Write a detailed PRD for [feature/product].
Include: problem statement, goals, non-goals, user personas, user stories,
success metrics, technical constraints, and open questions.
Format it with clear sections and use a decision-ready tone.
```

---

## 2. User Story Writing

```
Write user stories for [feature] targeting [user persona].
Format each as: As a [user type], I want to [action], so that [benefit].
Include acceptance criteria for each story.
Prioritize by user impact (High / Medium / Low).
```

---

## 3. Competitive Analysis

```
Conduct a competitive analysis for [product/feature] in the [industry] space.
Compare [Competitor A], [Competitor B], and [Competitor C] on:
- Core features
- Pricing model
- Target audience
- Strengths and weaknesses
- Gaps we could exploit
Output as a structured table, followed by a strategic summary.
```

---

## 4. User Research — Interview Script

```
Create a user research interview script for [product/feature].
The goal is to understand [user pain point or behavior].
Include: warm-up questions, core exploratory questions, follow-up probes,
and a closing question. Keep the tone conversational and neutral.
```

---

## 5. OKR Writing

```
Help me write OKRs for [team/quarter/goal].
Format:
- Objective: [Aspirational, qualitative goal]
  - KR1: [Measurable, time-bound result]
  - KR2: [Measurable, time-bound result]
  - KR3: [Measurable, time-bound result]
Make the KRs ambitious but achievable. Avoid output-based KRs.
```

---

## 6. Stakeholder Communication

```
Draft a stakeholder update email for [project/feature].
Status: [On track / At risk / Delayed]
Audience: [Executive / Engineering / Cross-functional]
Include: progress summary, key decisions made, blockers, and next steps.
Keep it concise — under 200 words. Use bullet points for clarity.
```

---

## 7. Feature Prioritization (RICE Framework)

```
Help me prioritize the following features using the RICE framework:
[List features]
For each, estimate:
- Reach (# users impacted per quarter)
- Impact (0.25 / 0.5 / 1 / 2 / 3)
- Confidence (%)
- Effort (person-weeks)
Calculate RICE score and rank them. Explain the top 3 choices.
```

---

## 8. Jobs-To-Be-Done (JTBD) Analysis

```
Apply the Jobs-To-Be-Done framework to [product/feature].
Identify:
- The functional job: what users are trying to accomplish
- The emotional job: how they want to feel
- The social job: how they want to be perceived
Then, map our current solution against each job dimension.
```

---

## 9. Go-To-Market (GTM) Strategy

```
Create a go-to-market plan for [product/feature] launching to [audience].
Include:
- Launch goals and success metrics
- Target customer segment
- Positioning and messaging
- Channel strategy (paid, organic, partnership)
- Launch timeline (pre-launch, launch day, post-launch)
- Risks and mitigation
```

---

## 10. Retrospective Facilitation

```
Design a 60-minute sprint retrospective for a team of [size].
Use the [Start / Stop / Continue] format.
Include: icebreaker, structured reflection activities, dot voting exercise,
action item capture, and a closing check-out. Make it engaging and time-boxed.
```

---

## 11. AI-Assisted Product Strategy

```
You are a product strategist. Given the following context about our product:
[Insert context]
Identify the top 3 strategic bets we should make over the next 12 months.
For each bet, explain: the opportunity, the risk, the required investment,
and the success signal. Be direct and opinionated.
```

---

## 12. Metrics Framework

```
Design a metrics framework for [feature/product].
Layer it as:
- North Star Metric: [Single most important metric]
- Level 1 (Outcome metrics): [3–4 metrics]
- Level 2 (Input/driver metrics): [5–6 metrics]
- Counter metrics: [2–3 guardrails]
Explain why each metric matters and how to measure it.
```

---

> 💡 **Tip:** These prompts work best when you add your specific product context, customer quotes, or data to the `[placeholders]`.
